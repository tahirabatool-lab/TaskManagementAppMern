📌 MERN Stack Todo Application
📖 Overview

This is a full-stack Todo application built using the MERN Stack (MongoDB, Express, React, Node.js).
It demonstrates a complete CRUD system where users can create, read, update, and delete tasks.

The project follows a client-server architecture. The frontend communicates with the backend using REST APIs, and all data is stored in MongoDB.

🎯 Project Purpose

This project is created to help understand:

Full-stack web development workflow
REST API development
Frontend and backend integration
MongoDB database operations
React state management
Real-world CRUD functionality
🏗️ Project Structure

The project contains two main parts:

Backend (Server)

Handles APIs, database connection, and server logic.
It includes models, routes, and server configuration.

Frontend (Client)

Handles user interface and user interaction.
It includes input fields, task display, and API communication.

⚙️ Features
Add new tasks
View all tasks
Edit existing tasks
Delete tasks
Real-time UI updates
REST API integration
MongoDB database storage
🔄 How It Works
User interacts with React frontend
Frontend sends request to backend API
Backend processes request using Express
MongoDB stores or retrieves data
Response is sent back to frontend
UI updates automatically
🛠️ Technologies Used
React.js
Node.js
Express.js
MongoDB
Axios
📦 How to Run This Project (GitHub / ZIP Download Guide)
Step 1: Clone or Download Project

You can either:

Clone the repository using Git
or
Download ZIP and extract it
Step 2: Open Project in VS Code

Extract the folder and open it in VS Code.

Step 3: Setup Backend
Open backend folder
Install dependencies
Start backend server
Ensure MongoDB is running locally or using MongoDB Atlas

Backend runs on:
http://localhost:5000

Step 4: Setup Frontend
Open client folder
Install dependencies
Start React development server

Frontend runs on:
http://localhost:3000

Step 5: Run Full Application
Start backend first
Then start frontend
Application will be fully functional
⚠️ Important Notes
MongoDB must be running before starting backend
Both frontend and backend must run at the same time
API URL must match backend server
